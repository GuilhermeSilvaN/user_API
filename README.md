# User_API

![UML do user](/img/users.png)
### Requisições web de usuário simples. <br>
> * Controller, responsável por receber as requisições e encaminhar para a camada service;<br>
> * Service, aplica as regras de negócios da aplicação, realizando validações e lógica;<br>
> * Repository, responsável pelo acesso e persitência dos dados;<br>
> * Database, camada responsável por armazenar os dados de forma persistente;

![Status](https://img.shields.io/badge/Status-Concluido-green)
![Java](https://img.shields.io/badge/Java-21-blue)

---
Uma API simples ao qual é possível fazer requisições simples com métodos HTTP; <br>
>   * GET
>   * POST
>   * PUT
>   * DELETE

## Arquitetura 
O modelo escolhido: **MVC (Model-View-Controller)**, perfeito para implementação de applicações web,
eficiência e utilização de metodologia SOLID.

## Tecnologias Utilizadas

* **Java 21**
* **IDE Intellij IDEA**
* **Banco de Dados: MySQL**
* **Spring Boot** utilizado para construir a aplicação web
* **JPA** (Java Persistence API) para mapear os objetos e relacioná-los no banco de dados
